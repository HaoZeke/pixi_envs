* What is Reference Documentation?

Reference documentation provides:
- Complete parameter listings with defaults
- Rule specifications and dependencies
- Output file formats
- Environment dependencies

Unlike tutorials, reference documentation:
- Does not teach concepts
- Assumes familiarity with the workflow
- Provides comprehensive, structured information
- Is organized by topic, not learning sequence

* Using This Section

- **Configuration**: All config.yaml parameters with defaults and guidelines
- **Rules**: Each Snakemake rule's inputs, outputs, and parameters
- **Environments**: Conda environment specifications per rule
- **Outputs**: File formats and interpretation
