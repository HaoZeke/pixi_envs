* Overview

{{workflow}} is a modular Snakemake workflow for automated Nudged Elastic Band (NEB)
calculations using eOn with machine learning potentials (PET-MAD/uPET).

The workflow orchestrates the complete NEB pipeline:
1. Automatic ML potential retrieval from HuggingFace
2. Endpoint preparation with IRA alignment and minimization
3. CI-NEB optimization with energy-weighted springs and MMF refinement
4. Publication-quality visualization (1D profiles, 2D landscapes)

** Key Features

- *Modular Design*: Each workflow stage is a separate, reusable rule module
- *Automated Dependencies*: Conda environments per rule for reproducibility
- *ML-Powered*: Uses PET-MAD/uPET machine learning potentials
- *Production-Ready*: Validated on molecular isomerization reactions
- *Extensible*: Easy to add new systems or modify parameters

** System Applicability

This workflow is optimized for =gas-phase molecular systems=:
- Small to medium organic molecules (5-50 atoms)
- Single or multiple bond breaking/forming reactions
- Isomerization and proton transfer reactions

*Note*: Surface reactions, condensed phase, or enzymatic systems require
modifications to cell parameters and boundary conditions.

* Installation

** Requirements

- Python 3.11+
- Snakemake 7.0+
- eOn client (for NEB calculations)
- CUDA-capable GPU (recommended) or CPU

* Usage

** Basic Workflow

#+begin_src bash
# 1. Copy and edit configuration
cp config/config.yaml my_config.yaml
# Edit my_config.yaml with your system paths

# 2. Run the complete workflow
snakemake --configfile my_config.yaml -c4 --use-conda

# 3. Check results
ls results/plots/*/
#+end_src

** Configuration

Edit =config/config.yaml= to define:

1. *Systems*: Reactant and product .con file paths
2. *Model*: PET-MAD version (default: v1.1.0)
3. *Compute*: Device (cuda/cpu)
4. *NEB Parameters*: Images, convergence, springs

Example minimal config:

#+begin_src yaml
systems:
  hcn_isom:
    reactant: /path/to/hcn.con
    product: /path/to/hnc.con
    use_ira: true

model:
  name: pet-mad-v1.1.0
  version: v1.1.0

compute:
  device: cuda
#+end_src

** Workflow Targets

#+begin_src bash
# Dry run (validate DAG)
snakemake -n --configfile my_config.yaml

# Run specific system
snakemake prepare_endpoints --wildcards system=hcn_isom --configfile my_config.yaml

# Run NEB only
snakemake run_neb_for_system --wildcards system=hcn_isom --configfile my_config.yaml

# Generate plots only
snakemake visualize_for_system --wildcards system=hcn_isom --configfile my_config.yaml

# Clean outputs
snakemake clean --configfile my_config.yaml
#+end_src

* Example Results

After running the workflow, outputs include:

** 1D Energy Profile

Energy along the reaction coordinate showing:
- Reactant and product basins
- Saddle point (transition state)
- Energy barrier height

** 2D RMSD Landscape

Projection of the path onto structural coordinates:
- X-axis: RMSD from reactant
- Y-axis: RMSD from product
- Color: Energy
- Reveals path tortuosity and intermediates
